File & Directory Management

10 commands

Core commands for navigating and managing the Linux filesystem — essential for every DevOps engineer.

ls
📌 Purpose

List files and directories. Use -la to reveal hidden files and permissions — critical when debugging deployment artifacts or config files.

Syntax
ls [options] [directory]
Examples
ls -la /var/log
ls -lh --sort=size /opt/app
cd
📌 Purpose

Change the current working directory. Navigate between project repos, log directories, and config paths.

Syntax
cd [directory]
Examples
cd /etc/nginx
cd ~/projects/app
cd - # go back to previous dir
pwd
📌 Purpose

Print working directory. Confirm your current path, especially inside shell scripts or after long navigation chains.

Syntax
pwd
Examples
pwd
echo "Current path: $(pwd)"
mkdir
📌 Purpose

Create directories. Use -p to create nested paths in one shot — essential when provisioning app directory trees.

Syntax
mkdir [options] directory
Examples
mkdir -p /opt/app/{logs,config,data}
mkdir -m 755 /srv/deployments
rm
📌 Purpose

Remove files or directories. Use with caution in production. -rf removes recursively — always double-check paths before running.

Syntax
rm [options] file/directory
Examples
rm -rf /tmp/build-artifacts
rm -i important.conf # interactive confirm
cp
📌 Purpose

Copy files or directories. Use -r for directories and -p to preserve timestamps and permissions — important for backup scripts.

Syntax
cp [options] source destination
Examples
cp -rp /opt/app /opt/app.bak
cp nginx.conf nginx.conf.orig
mv
📌 Purpose

Move or rename files and directories. Used in deployment scripts to atomically swap config files or promote build outputs.

Syntax
mv [options] source destination
Examples
mv app.jar.new app.jar
mv /tmp/release /opt/app/v2.1
find
📌 Purpose

Search for files based on name, type, size, age, or permissions. Invaluable for auditing systems and cleanup automation.

Syntax
find [path] [options] [expression]
Examples
find /var/log -name '*.log' -mtime +7 -delete
find / -perm -4000 -type f 2>/dev/null
touch
📌 Purpose

Create an empty file or update a file's timestamp. Used to create lock files, trigger watchers, or initialize config placeholders.

Syntax
touch [options] file
Examples
touch /tmp/deploy.lock
touch -t 202401011200 file.txt
ln
📌 Purpose

Create hard or symbolic links. Symlinks are used extensively in config management (e.g., Nginx sites-enabled, alternatives system).

Syntax
ln [options] target link_name
Examples
ln -s /opt/app/current /opt/app/latest
ln -sf /etc/nginx/sites-available/app /etc/nginx/sites-enabled/app

File Permissions & Ownership

6 commands

Control who can read, write, and execute files. Critical for securing deployments and running services under correct user contexts.

chmod
📌 Purpose

Change file permissions using octal or symbolic notation. Misconfigured permissions are a common cause of deployment failures.

Syntax
chmod [options] mode file
Examples
chmod 755 /opt/app/bin/start.sh
chmod -R 644 /var/www/html
chmod u+x deploy.sh
chown
📌 Purpose

Change file ownership. Services should run as dedicated non-root users — chown enforces that ownership model.

Syntax
chown [options] user[:group] file
Examples
chown -R appuser:appgroup /opt/app
chown nginx:nginx /var/log/nginx
chgrp
📌 Purpose

Change the group ownership of a file. Useful when multiple services share access to a common resource via group membership.

Syntax
chgrp [options] group file
Examples
chgrp -R www-data /var/www
chgrp docker /var/run/docker.sock
umask
📌 Purpose

Set the default permission mask for newly created files. Used in shell profiles and service startup scripts to enforce secure defaults.

Syntax
umask [mask]
Examples
umask 022 # new files: 644, dirs: 755
umask 027 # stricter: new files: 640
stat
📌 Purpose

Display detailed file metadata including permissions, inode, owner, timestamps. Invaluable for debugging permission and config issues.

Syntax
stat [options] file
Examples
stat /etc/passwd
stat -c '%A %U %G' /opt/app/config.yaml
getfacl / setfacl
📌 Purpose

Get and set POSIX Access Control Lists — granular permission control beyond the traditional user/group/other model.

Syntax
getfacl file
setfacl [options] file
Examples
getfacl /opt/app/data
setfacl -m u:deploy:rw /opt/shared/config

Process Management

8 commands

Monitor, control, and manage running processes. Essential for debugging hung services, resource contention, and graceful deployments.

ps
📌 Purpose

Snapshot of current running processes. ps aux is the classic DevOps command to identify what's consuming resources or confirm a service started.

Syntax
ps [options]
Examples
ps aux | grep java
ps -ef --sort=-%cpu | head -10
top / htop
📌 Purpose

Real-time process monitor. htop provides a colorized, interactive view. Essential for diagnosing CPU/memory spikes during load tests.

Syntax
top [options]
htop
Examples
top -bn1 | head -20 # non-interactive snapshot
htop -u appuser # filter by user
kill
📌 Purpose

Send signals to processes by PID. SIGTERM (15) for graceful shutdown; SIGKILL (9) for forced termination.

Syntax
kill [signal] PID
Examples
kill -15 1234 # graceful stop
kill -9 1234 # force kill
kill -HUP $(cat /var/run/nginx.pid)
pkill
📌 Purpose

Kill processes by pattern match on name or other attributes. More flexible than killall for scripted process management.

Syntax
pkill [options] pattern
Examples
pkill -f 'python app.py'
pkill -u appuser # kill all processes by user
nohup
📌 Purpose

Run a command immune to hangups (SIGHUP). Keeps processes running after SSH session ends.

Syntax
nohup command [args] &
Examples
nohup ./start-worker.sh > worker.log 2>&1 &
nohup python3 app.py &
nice / renice
📌 Purpose

Start or change process CPU priority. Used to deprioritize background batch jobs during peak hours.

Syntax
nice -n value command
renice -n value -p PID
Examples
nice -n 19 ./backup.sh
renice -n 5 -p 3456
lsof
📌 Purpose

List open files and network connections by processes. Critical for diagnosing 'port already in use' errors.

Syntax
lsof [options]
Examples
lsof -i :8080 # who's using port 8080
lsof -p 1234 # files opened by PID 1234
strace
📌 Purpose

Trace system calls made by a process. Powerful debugging tool to understand why a program fails — shows every file open, network call, and syscall.

Syntax
strace [options] command
Examples
strace -p 1234 # attach to running process
strace -e trace=network curl example.com
strace -o trace.log ./app

System Monitoring

8 commands

Observe resource utilization, uptime, and hardware health to keep production systems stable.

vmstat
📌 Purpose

Report virtual memory, CPU, I/O statistics. Use for identifying memory pressure and CPU wait times affecting application performance.

Syntax
vmstat [options] [delay] [count]
Examples
vmstat 2 5 # 5 snapshots, 2s apart
vmstat -s # summary stats
iostat
📌 Purpose

Show CPU and disk I/O statistics. Identify disk bottlenecks causing slow reads/writes in database or file-heavy applications.

Syntax
iostat [options] [delay] [count]
Examples
iostat -xz 1 5
iostat -d -h /dev/sda
free
📌 Purpose

Display memory usage (RAM + swap). Essential health check — low free memory often explains application slowdowns or OOM kills.

Syntax
free [options]
Examples
free -h # human readable
free -m -s 3 # update every 3 seconds
uptime
📌 Purpose

Show system uptime and load averages (1, 5, 15 minutes). High load average relative to CPU count indicates a saturated system.

Syntax
uptime
Examples
uptime
cat /proc/loadavg
df
📌 Purpose

Show disk filesystem usage. Full disks cause application crashes — always monitor disk space, especially log and data volumes.

Syntax
df [options] [filesystem]
Examples
df -h
df -h /var/log
df -i # inode usage
du
📌 Purpose

Estimate disk usage of files and directories. Use to find what's consuming disk space when a volume fills unexpectedly.

Syntax
du [options] [directory]
Examples
du -sh /var/log/*
du -h --max-depth=1 /opt | sort -h
dmesg
📌 Purpose

Print kernel ring buffer messages. First stop for diagnosing hardware errors, OOM kills, disk failures, and kernel panics.

Syntax
dmesg [options]
Examples
dmesg | tail -50
dmesg -T | grep -i error
dmesg --level=err,warn
watch
📌 Purpose

Execute a command repeatedly and display output. Great for monitoring changing metrics without writing a loop.

Syntax
watch [options] command
Examples
watch -n 2 'df -h'
watch -d 'cat /proc/loadavg' # highlight changes

Networking

10 commands

Configure, test, and diagnose network connectivity — the backbone of all distributed DevOps infrastructure.

ip
📌 Purpose

Modern replacement for ifconfig/route. View and configure network interfaces, routes, and addresses. The go-to tool for network config in modern Linux.

Syntax
ip [options] object command
Examples
ip addr show
ip route show
ip link set eth0 up
ss
📌 Purpose

Socket statistics — faster, more detailed replacement for netstat. Check listening ports, established connections, and socket states.

Syntax
ss [options]
Examples
ss -tulnp # listening TCP/UDP with process
ss -s # summary
ss -t state established
ping
📌 Purpose

Test ICMP reachability to a host. Basic connectivity check to verify network paths and measure round-trip latency.

Syntax
ping [options] host
Examples
ping -c 4 google.com
ping -i 0.2 -c 100 192.168.1.1
traceroute
📌 Purpose

Trace the network path to a destination. Identify where packets are dropped or slow — essential for network troubleshooting.

Syntax
traceroute [options] host
Examples
traceroute -n google.com
tracepath api.example.com
dig
📌 Purpose

DNS lookup tool. Verify DNS records, propagation, and resolver behavior — critical when deploying new services or debugging DNS issues.

Syntax
dig [options] name [type]
Examples
dig api.example.com A
dig @8.8.8.8 example.com MX
dig +short example.com
curl
📌 Purpose

Transfer data to/from URLs. The Swiss Army knife of HTTP testing — health checks, API calls, file downloads, header inspection.

Syntax
curl [options] URL
Examples
curl -I https://api.example.com/health
curl -X POST -H 'Content-Type: application/json' -d '{"key":"val"}' https://api.example.com
curl -o output.tar.gz https://releases.example.com/v1.2.tar.gz
wget
📌 Purpose

Non-interactive file downloader. Supports recursive downloads and resuming interrupted transfers.

Syntax
wget [options] URL
Examples
wget -q -O /tmp/app.jar https://releases.example.com/app-1.0.jar
wget -c --retry-connrefused https://large-file.example.com/data.zip
nmap
📌 Purpose

Network port scanner. Discover open ports and services on remote hosts — essential for security audits and firewall validation.

Syntax
nmap [options] target
Examples
nmap -sV -p 22,80,443 192.168.1.0/24
nmap -A --script=default 10.0.0.1
tcpdump
📌 Purpose

Capture and analyze network packets. Diagnose connection issues and validate TLS/HTTP communication at the packet level.

Syntax
tcpdump [options]
Examples
tcpdump -i eth0 -n port 443
tcpdump -i any host 10.0.0.5 -w capture.pcap
iptables
📌 Purpose

Linux firewall management. Control packet filtering, NAT, and port forwarding. nftables is the modern successor.

Syntax
iptables [options]
Examples
iptables -L -n -v
iptables -A INPUT -p tcp --dport 8080 -j ACCEPT
nft list ruleset

Disk & Storage

6 commands

Manage block devices, filesystems, volumes, and storage — critical for persistent data management in production environments.

lsblk
📌 Purpose

List block devices in a tree view. Quickly see disks, partitions, and mount points — essential when provisioning storage.

Syntax
lsblk [options]
Examples
lsblk
lsblk -f # show filesystems
fdisk / parted
📌 Purpose

Create and manage disk partitions. fdisk is classic for MBR; parted handles both MBR and GPT and is required for disks >2TB.

Syntax
fdisk [options] device
parted [device]
Examples
fdisk -l
parted /dev/sdb print
parted /dev/sdb mklabel gpt
mkfs
📌 Purpose

Format a partition with a filesystem. Use after creating a partition before mounting — choose ext4 for general use, xfs for large files.

Syntax
mkfs.[type] [options] device
Examples
mkfs.ext4 /dev/sdb1
mkfs.xfs /dev/sdc1
mount / umount
📌 Purpose

Attach or detach filesystems. Mount remote NFS shares, loop devices, or new disks to specific directories.

Syntax
mount [options] device mountpoint
Examples
mount /dev/sdb1 /mnt/data
mount -t nfs 10.0.0.5:/exports /mnt/nfs
umount /mnt/data
dd
📌 Purpose

Low-level copy of raw data. Used for disk imaging, benchmarking I/O, creating swap files, and wiping disks. Double-check of= before running.

Syntax
dd [operands]
Examples
dd if=/dev/zero of=/swapfile bs=1M count=2048
dd if=/dev/sda of=/backup/disk.img bs=4M status=progress
lvextend / resize2fs
📌 Purpose

Extend a logical volume and then resize the filesystem. Expand disk space for growing application data volumes without downtime.

Syntax
lvextend -L +size LV
resize2fs device
Examples
lvextend -L +20G /dev/vg0/data
resize2fs /dev/vg0/data

Package Management

5 commands

Install, update, and remove software packages. Package managers vary by distribution — knowing the right tool is essential for provisioning and automation.

apt / apt-get
📌 Purpose

Debian/Ubuntu package manager. Used in most containerized and cloud Linux environments. apt is user-friendly; apt-get is preferred in scripts.

Syntax
apt [options] command
Examples
apt update && apt upgrade -y
apt install -y curl wget git
apt-get install --no-install-recommends nginx
yum / dnf
📌 Purpose

RHEL/CentOS/Fedora package manager. dnf is the modern replacement for yum. Essential for RHEL-based enterprise and cloud environments.

Syntax
dnf [options] command
Examples
dnf update -y
yum install -y java-17-openjdk
dnf install --setopt=install_weak_deps=False nginx
rpm
📌 Purpose

Low-level RPM package tool. Install, query, verify, and remove .rpm packages directly.

Syntax
rpm [options] package
Examples
rpm -ivh package.rpm
rpm -qa | grep java
rpm -qf /usr/bin/python3 # which package owns file
dpkg
📌 Purpose

Debian low-level package tool. Directly install or query .deb packages. Used in Docker builds and CI pipelines.

Syntax
dpkg [options] package
Examples
dpkg -i package.deb
dpkg -l | grep nginx
dpkg -L nginx # list files installed by package
pip3
📌 Purpose

Python package installer. Critical in DevOps for installing Ansible, AWS CLI, linting tools, and other Python-based tooling.

Syntax
pip3 [options] command
Examples
pip3 install ansible boto3
pip3 install -r requirements.txt
pip3 list --outdated

Users & Groups

6 commands

Manage user accounts and group memberships — foundational for role-based access control (RBAC) on Linux servers.

useradd
📌 Purpose

Create new user accounts. Use for creating service accounts in provisioning scripts.

Syntax
useradd [options] username
Examples
useradd -m -s /bin/bash -G docker appuser
useradd -r -s /sbin/nologin -d /opt/app appservice # system account
usermod
📌 Purpose

Modify an existing user's properties — groups, shell, home directory. Commonly used to add users to the docker or sudo group.

Syntax
usermod [options] username
Examples
usermod -aG docker ubuntu
usermod -aG sudo,adm deployuser
passwd
📌 Purpose

Set or change user passwords. Also used to lock/unlock accounts. Prefer SSH keys over passwords for service accounts.

Syntax
passwd [options] [username]
Examples
passwd deploy
passwd -l serviceacct # lock account
echo 'user:password' | chpasswd # non-interactive
id
📌 Purpose

Show user and group identity (UID/GID). Quickly verify which groups a user belongs to — important for confirming docker/sudo access.

Syntax
id [username]
Examples
id
id ubuntu # check another user
id -nG appuser # list group names only
sudo / su
📌 Purpose

Execute commands as another user. sudo is the standard for privilege escalation — always audit sudoers file in production.

Syntax
sudo [options] command
su [options] [username]
Examples
sudo systemctl restart nginx
sudo -u appuser ./run.sh
su - postgres
who / w / last
📌 Purpose

Show currently logged-in users and login history. Essential for security auditing and tracking unauthorized access.

Syntax
who
w
last [username]
Examples
who
w # detailed with what they're running
last -20 # last 20 logins

Logs & Troubleshooting

5 commands

Analyze and follow logs to diagnose incidents in real time — the most critical skill for production support and SRE work.

tail
📌 Purpose

Output the last N lines of a file. tail -f follows a log in real time — the most-used command during deployments and incident response.

Syntax
tail [options] file
Examples
tail -f /var/log/nginx/error.log
tail -n 200 /var/log/app/app.log
tail -f /var/log/syslog | grep ERROR
journalctl
📌 Purpose

Query systemd's centralized journal. The primary tool for reading service logs on modern Linux — supports filtering by unit, time, and priority.

Syntax
journalctl [options]
Examples
journalctl -u nginx -f # follow nginx logs
journalctl --since '1 hour ago' -p err
journalctl -u app.service --no-pager | tail -100
grep
📌 Purpose

Search text using patterns. Combine with tail/cat to filter logs for specific errors, IP addresses, or request IDs.

Syntax
grep [options] pattern file
Examples
grep -i 'error|exception' /var/log/app.log
grep -n 'FATAL' app.log | tail -20
journalctl -u app | grep '500'
less
📌 Purpose

Page through file contents. Allows bidirectional scrolling. Essential for reading large log files without loading them into memory.

Syntax
less [options] file
Examples
less +F /var/log/syslog # follow mode like tail -f
less -S /var/log/large.log # no line wrap
logger
📌 Purpose

Write messages to the system log. Use in shell scripts to record deployment events and errors into syslog for centralized monitoring.

Syntax
logger [options] message
Examples
logger -t deploy 'Deployment started for v1.2.3'
logger -p local0.info 'Backup completed successfully'

Text Processing

7 commands

Parse, transform, and extract data from text files and command output — essential for log analysis and config generation.

awk
📌 Purpose

Pattern scanning and text processing language. Extract columns, calculate sums, and reformat structured text output.

Syntax
awk [options] 'program' file
Examples
awk '{print $1, $4}' access.log
awk -F: '{print $1}' /etc/passwd # list usernames
df -h | awk 'NR>1 {print $5, $6}'
sed
📌 Purpose

Stream editor for in-place text substitution and deletion. Ideal for modifying config files and replacing placeholders in templates.

Syntax
sed [options] 'expression' file
Examples
sed -i 's/localhost/db.prod.example.com/g' app.conf
sed -n '10,50p' large.log # print lines 10-50
sed '/^#/d' config.conf # remove comment lines
cut
📌 Purpose

Cut sections from lines of files. Extract delimited fields from CSV-like data or command output quickly.

Syntax
cut [options] file
Examples
cut -d: -f1,3 /etc/passwd
echo '2024-01-15 INFO Starting' | cut -d' ' -f3-
sort
📌 Purpose

Sort lines of text. Use with uniq to count occurrences in logs, or with -k and -n for numeric/column-based sorting.

Syntax
sort [options] file
Examples
sort -k5 -n -r access.log | head
sort -u file.txt # sort + deduplicate
uniq
📌 Purpose

Filter or report duplicate lines. Always pipe sorted input to uniq. Use -c to count occurrences — great for log frequency analysis.

Syntax
uniq [options] [input]
Examples
sort errors.log | uniq -c | sort -rn | head -20
cat ips.txt | sort | uniq -d # show duplicate IPs
wc
📌 Purpose

Count words, lines, characters. Use wc -l to count lines in a file — common in scripts to verify record counts after processing.

Syntax
wc [options] file
Examples
wc -l /etc/hosts
grep 'ERROR' app.log | wc -l # count error lines
tr
📌 Purpose

Translate or delete characters. Use for case conversion, removing carriage returns from Windows files, or collapsing whitespace.

Syntax
tr [options] SET1 [SET2]
Examples
echo 'Hello World' | tr '[:upper:]' '[:lower:]'
cat windows.txt | tr -d '\r' > unix.txt

Searching & Filtering

5 commands

Locate files, patterns, and content efficiently across filesystems and data streams.

grep -r
📌 Purpose

Recursively search for patterns across all files in a directory tree. Essential for finding config values, secrets, or code patterns across a project.

Syntax
grep -r [options] pattern directory
Examples
grep -r 'database_url' /etc/app/
grep -rn 'TODO' ./src --include='*.py'
find
📌 Purpose

Search for files by name, type, size, time, or permissions. Supports -exec to act on results — more powerful than locate for scripting.

Syntax
find path [expression]
Examples
find /opt -name '*.conf' -newer /opt/last-deploy
find /tmp -type f -mmin +60 -delete
find / -user root -perm -4000 -print # SUID files
locate
📌 Purpose

Fast file search using a pre-built database index. Much faster than find for name searches — run updatedb to refresh the index.

Syntax
locate [options] pattern
Examples
locate nginx.conf
locate -i '*.jar' # case-insensitive
which / whereis
📌 Purpose

Find the location of executable commands. Essential for confirming which binary will run in your PATH when multiple versions coexist.

Syntax
which command
whereis command
Examples
which python3
whereis java
type kubectl
xargs
📌 Purpose

Build and execute commands from standard input. Chain with find or grep to perform bulk operations on matched files.

Syntax
xargs [options] command
Examples
find /opt -name '*.log' | xargs rm -f
grep -rl 'old-host' /etc/ | xargs sed -i 's/old-host/new-host/g'

Archiving & Compression

4 commands

Pack, compress, and extract files for deployments, backups, and artifact transfers.

tar
📌 Purpose

Tape archive tool — pack multiple files into a single archive. The standard for distributing application artifacts and creating backups in Linux.

Syntax
tar [options] archive [files]
Examples
tar -czvf app-v1.2.tar.gz /opt/app/
tar -xzvf release.tar.gz -C /opt/
tar -tzvf archive.tar.gz # list contents without extracting
gzip / gunzip
📌 Purpose

Compress or decompress individual files using the gzip algorithm. Most commonly used with tar (tar -z).

Syntax
gzip [options] file
gunzip file.gz
Examples
gzip -k -9 large-log.log # keep original, max compression
gunzip -c data.gz | grep 'ERROR'
zip / unzip
📌 Purpose

Cross-platform archive format. Required when creating archives for Windows users or tools that expect .zip (e.g., AWS Lambda deployments).

Syntax
zip [options] archive.zip files
unzip archive.zip
Examples
zip -r lambda-deploy.zip ./function/ -x '*.git*'
unzip -o release.zip -d /opt/app/
rsync
📌 Purpose

Efficient file synchronization with delta transfer. Essential for deployments, backups, and syncing across servers — only transfers changed bytes.

Syntax
rsync [options] source destination
Examples
rsync -avz --delete /opt/app/ user@prod:/opt/app/
rsync -avzP --exclude='.git' ./ server:/opt/project/

SSH & Remote Access

5 commands

Securely connect to and manage remote servers — the primary interface for cloud infrastructure management.

ssh
📌 Purpose

Open Secure Shell connections to remote hosts. The primary way to access cloud instances, jump hosts, and remote servers.

Syntax
ssh [options] user@host
Examples
ssh ubuntu@10.0.0.5
ssh -i ~/.ssh/deploy.pem ec2-user@ec2-54-x.compute.amazonaws.com
ssh -L 8080:localhost:8080 user@bastion.example.com # port forward
ssh-keygen
📌 Purpose

Generate SSH key pairs. Create ed25519 keys for secure, passwordless authentication to servers and CI/CD systems.

Syntax
ssh-keygen [options]
Examples
ssh-keygen -t ed25519 -C 'deploy@ci' -f ~/.ssh/deploy_key
ssh-keygen -t rsa -b 4096 -N '' # no passphrase for automation
ssh-copy-id
📌 Purpose

Copy SSH public key to remote host's authorized_keys. Simplifies setting up passwordless SSH access without manually editing files.

Syntax
ssh-copy-id [options] user@host
Examples
ssh-copy-id -i ~/.ssh/id_ed25519.pub ubuntu@192.168.1.5
scp
📌 Purpose

Securely copy files between hosts using SSH. Use for one-off file transfers — for repetitive sync, prefer rsync.

Syntax
scp [options] source user@host:dest
Examples
